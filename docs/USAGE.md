# 使用说明

本文档面向实际使用者，按“从零开始到完成一轮比赛数据处理”的顺序说明。

项目目录：

`C:\Users\Theta\uestc_tt_manager`

## 一、启动系统

### 1. 激活虚拟环境

```powershell
cd C:\Users\Theta\uestc_tt_manager
.\.venv\Scripts\Activate.ps1
```

如果你还没有虚拟环境，先执行：

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
playwright install chromium
```

### 2. 启动 Web 服务

```powershell
uvicorn app.main:app --host 127.0.0.1 --port 8050 --reload
```

或者直接双击：

`start_server.bat`

### 3. 打开页面

浏览器访问：

[http://127.0.0.1:8050](http://127.0.0.1:8050)

## 二、系统配置

在页面顶部“系统配置”区域完成以下设置。

### 1. 成电杯赛事群群号

填写后，后续 QQ 预检与群临时会话通知都会默认使用这个群。

### 2. NapCat 配置路径

默认通常类似：

`C:\ProgramData\NapCatQQ Desktop\runtime\NapCatQQ\config\onebot11_3496930386.json`

### 3. 金数据主页

默认保留：

`https://jinshuju.net/home`

### 4. 金数据 Profile 目录

这里保存 Playwright 浏览器登录态。第一次登录成功后，后续导出会复用这个目录。

### 5. 通知模板

通知模板支持 `{name}` 占位符。

示例：

```text
{name}同学你好，我是乒协机器人，这边发现你在正赛资格名单中，但单打项目报名表中没有你的信息。请你确认是否参加成电杯单打项目，如果参加，请填写群里的单打报名链接。
```

### 6. 刷新金数据表单列表

点击：

- `检查金数据登录`
- 或 `登录并刷新表单列表`

如果浏览器弹出金数据登录页，就在浏览器中完成登录。

### 7. 绑定表单

需要绑定四张表：

- 资格赛报名表
- 正赛单打报名表
- 正赛双打报名表
- 正赛团体报名表

推荐做法：

1. 从下拉框中直接选表
2. 检查标题和 `entries_url` 是否回填
3. 点击 `保存配置`

## 三、维护本地资格名单

这一部分的目标是生成和更新本地主表：

`data/qualification_master.xlsx`

### 1. 输入新增资格人员姓名

在“资格名单维护”区域，把新增资格人员姓名按“一行一个”粘贴进去。

例如：

```text
张三
李四
王五
```

### 2. 从资格赛报名表匹配 QQ

点击：

`从资格赛报名表匹配 QQ`

系统会：

1. 导出资格赛金数据表
2. 从导出表中读取 `姓名 / 学院 / QQ号`
3. 逐个按姓名匹配

### 3. 处理匹配结果

结果分为三类：

- 已直接匹配
- 待人工确认
- 未匹配

如果有同名多条，页面会给出候选项，需要你手动选定。

### 4. 合并写入本地资格名单

点击：

`合并更新到本地资格名单`

更新规则：

- 优先按 `QQ号` 合并
- 如果 `QQ号` 缺失，再按 `姓名 + 学院` 合并

### 5. 刷新与导出主表

可以使用：

- `刷新本地资格名单`
- `导出本地资格名单 Excel`

还可以使用：

- `收起已获取资格名单`
- `展开已获取资格名单`

## 四、单打报名比对

这部分主要有两条比对链路。

### 1. 找“报了单打，但没在资格赛报名表里出现”的人

点击：

`比对：单打报名但没在资格赛报名表出现`

系统会：

1. 导出资格赛报名表
2. 导出正赛单打报名表
3. 统一标准化 `姓名 / 学院 / QQ号`
4. 做比对

输出在：

`runs/compare-singles-qualification-form-时间戳/`

重点文件：

- `registered_singles_not_in_qualification_form.csv`
- `matched.csv`
- `summary.json`

### 2. 找“有资格，但没报单打”的人

点击：

`比对：有资格但没报单打`

系统会：

1. 读取本地资格名单主表
2. 导出最新单打报名表
3. 做比对

输出在：

`runs/compare-qualified-singles-时间戳/`

重点文件：

- `qualified_not_registered_singles.csv`
- `matched.csv`
- `summary.json`

## 五、QQ 预检与通知

只有在完成“有资格但没报单打”比对之后，才建议执行这一步。

### 1. 预检

点击：

`对“有资格但没报单打”名单做 QQ 预检`

系统会检查：

- NapCat 配置是否可读
- OneBot 连接是否正常
- 是否能在配置好的赛事群里找到对应成员
- 是否可发群临时会话

### 2. 查看预检结果

页面会给出通知候选表格，包含：

- 姓名
- 学院
- QQ号
- 预检状态
- 错误信息
- 消息预览

默认只勾选：

- `precheck_ok`

### 3. 发送通知

点击：

`给勾选对象发通知`

如果你没有勾选任何人，系统会提示先勾选。

## 六、优赛上传表生成

这一部分用于最终整理单打、双打、团体报名数据。

### 1. 提供最终报名文件

有两种方式：

- 直接填写本地文件路径
- 使用页面上传文件

需要提供三份文件：

- 单打
- 双打
- 团体

### 2. 预处理与查重

点击：

`查重并生成 sheet_output`

系统会：

1. 读取三份文件
2. 调用 vendored 转换器
3. 生成中间结果
4. 解析重复项

### 3. 确认删除重复项

点击：

`确认删除重复项并生成优赛上传表`

当前固定规则：

- 保留每组重复中的第一条或第一组

### 4. 输出文件

输出目录一般在：

`runs/converter-prepare-时间戳/`

或对应的确认目录中。

重点文件包括：

- `sheet_output.xlsx`
- `duplicate_review.csv`
- `duplicate_summary.json`
- `upload_preview.xlsx`
- `优赛上传表.xlsx`

## 七、常见故障排查

### 1. 金数据导出失败

先检查：

- 是否已登录金数据
- 当前绑定的表单是否正确
- 浏览器 Profile 是否还是同一个

如果系统提示调试文件路径，可以去对应 `runs/.../exports/.../` 目录看 HTML 和截图。

### 2. 提示报名表缺少列

当前系统已兼容常见列：

- `姓名`
- `QQ号`
- `学院`
- `学院部门（教学科研单位、研究机构）`

如果你的表单又换了表头，需要继续补规则。

### 3. QQ 预检没有勾选框

刷新页面后重新做一遍预检，表格才会生成可勾选行。

### 4. 发送通知时提示“请先勾选”

说明当前没有任何一行被勾选，或者预检结果没有 `precheck_ok` 的对象。

### 5. 匹配 QQ 按钮点了没反应

现在页面会把错误直接显示在结果框里。

先看：

- 匹配结果区
- 顶部状态栏
- 后端控制台

## 八、建议备份

建议定期备份：

- `data/qualification_master.xlsx`
- `data/config.json`

如果你已经跑过多轮任务，也可以按需备份：

- `runs/`

## 九、建议不要上传到公开仓库的内容

如果你要把项目公开到 GitHub，请不要上传：

- `browser_state/`
- `runs/`
- `data/config.json`
- `data/qualification_master.xlsx`
- 任意真实报名数据
- NapCat token 或 QQ 配置敏感信息

当前 `.gitignore` 已默认排除上述大部分本地数据。
